# Student Portal & Hostel Services System

## Overview
This project is a self-service web portal designed to digitalize student-related workflows. It eliminates manual paperwork and physical office visits by allowing students to securely access academic services and hostel facilities online.

**Course:** CS 345 / CS 346  
**Instructor:** Prof. Pradeep K. Das  

## Core Modules
1. **Authentication:** Secure student login via institute ID, password reset, and session management.
2. **Course Management:** Online course registration, add/drop functionality, and enrolled course tracking.
3. **Academic Records:** Access to grades, attendance reports, and transcript request status.
4. **Assignment Portal:** Document upload functionality with strict deadline enforcement and feedback viewing.
5. **Hostel Services:** Automated forms for leave applications, monthly mess subscription changes, and transfer requests.
6. **Complaint System:** Portal to raise, track, and receive notifications for issues regarding infrastructure, gym, and sports equipment.

## Team Collaboration & Git Workflow
To maintain code quality and prevent merge conflicts, all team members must follow this strict Pull Request workflow:

1. **Never push directly to the `main` branch.**
2. Create a new branch for your specific task: `git checkout -b feature/your-feature-name`
3. Commit and push your changes to your branch: `git push origin feature/your-feature-name`
4. Open a **Pull Request (PR)** on GitHub.
5. At least one other team member must review and approve the PR before it can be merged into `main`.
